# Markdown 渲染验收总览

这是一篇专门用于验收文章详情页渲染效果的测试文章。它的目标不是表达某个主题，而是**尽可能完整地覆盖常见 Markdown 元素**，让你能快速判断当前前端样式是否存在问题。

你可以重点观察以下内容是否被正确渲染：

- 一级、二级、三级标题是否有明显层级
- 正文段落的行高、留白、对齐是否舒服
- 列表、引用、代码块、图片、表格是否有独立样式
- 行内代码、粗体、斜体、链接是否可辨识

---

## 一、标题层级测试

这一节用于测试标题系统是否工作正常。你应该能明显看出 `h1`、`h2`、`h3` 的视觉差异。

这是 `h3` 下的一段文字。正常情况下，它应该比 `h2` 更小，但仍保有清晰的小节层级感，而不是退化成普通段落。

### 再来一个三级标题

如果你的样式正常，这里的标题间距、正文留白、标题与段落之间的节奏应该是稳定的，不会显得拥挤，也不会显得太散。

---

## 二、正文与强调样式测试

这是普通段落文本，用来观察正文的基础阅读体验。理想状态下，你应该能感受到舒适的行高、适度的段落留白，以及清晰但不过分刺眼的文字颜色。

这段里包含 **粗体文本**、*斜体文本*、***粗斜体文本***，还包含 `行内代码`，用于测试行内强调元素是否有足够的区分度。

你还可以检查一个链接的显示效果，例如访问 [Next.js 官网](https://nextjs.org/) 看看链接颜色、下划线、悬停反馈是否自然。

```ts
}

export function renderArticle(meta: ArticleMeta, mode: RenderMode) {
  return {
    ...meta,
    mode,
    status: "ready"
  };
}
```

下面是一个 Bash 代码块：


如果你看到代码块顶部有状态栏、语言标识和复制按钮，说明这部分渲染链路已经接通了。

## 九、混合内容测试

下面是一段混合内容，专门用来测试复杂排版是否会乱：

在一个真实的技术文章里，我们可能会同时出现 **重点结论**、`配置项名称`、[官方文档链接](https://react.dev/)、列表说明，以及引用块或代码块。如果这些内容在同一页里都能保持稳定的视觉节奏，那就说明当前文章页的 Markdown 渲染已经基本可用了。
